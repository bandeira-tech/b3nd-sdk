Data belongs to you. Four operations. Any app.

The Whole Protocol

Everything B3nd does, on one screen.

receive

Write data to a URI

read

Get data from a URI

list

Browse URIs by prefix

delete

Remove data at a URI

mutable://

Updatable data. Profiles, configs, state.

immutable://

Write-once records. Logs, posts, receipts.

blob://

Content-addressed by SHA256 hash.

link://

Mutable references to other URIs.

msg://

Message envelopes for batch writes.

Show Don't Tell

write 
// Store data at a URI you control
await client.receive([
  "mutable://accounts/alice/profile",
  { name: "Alice", bio: "Builder" }
]);
read 
// Read it back
const result = await client.read(
  "mutable://accounts/alice/profile"
);

console.log(result.record.data);
// { name: "Alice", bio: "Builder" }
any app 
// A completely different app reads the same data
const profile = await otherApp.read(
  "mutable://accounts/alice/profile"
);

// Same URI, same data. Your data travels with you.

Your Data, Your Keys

Today

B3nd

Traditional platforms sit between users and their data. B3nd puts data at the center — any app can read the same addresses, and switching apps doesn't mean starting over.

Privacy is Math

Public

No encryption. Open data anyone can read.

Protected

Password-derived key. Share the password, share access.

Private

Recipient's public key. Only the intended reader can decrypt.
